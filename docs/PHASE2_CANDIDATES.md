# PHASE2_CANDIDATES.md — 미디어팀 Phase 2 검토 후보

> **작성**: 2026-05-31 (Phase 1 종결 시점)
> **상태**: 검토 대기 — 대표가 실전 운영 후 우선순위 결정
> **연관**: `docs/PLAN.md` §3.7(Phase 분할), `docs/CHECKLIST.md` 보류·차기

Phase 1은 "재료 자동 생성 + 5분 수동 편집" 구조로 완주했다. Phase 2는 **수동 단계를 줄이는 자동화** 3가지를 후보로 둔다. 셋 다 Phase 1을 안 건드리고 **추가**되는 모듈이라, 하나씩 독립 도입 가능.

---

## 후보 1. Remotion — 자동 영상 렌더

| 항목 | 내용 |
|------|------|
| **무엇** | React 코드로 영상(MP4)을 100% 자동 렌더. `vrew_script.txt`/`script.md`를 입력으로 컷·자막·전환을 코드 생성. |
| **해결** | 현재 "Vrew/CapCut 5분 편집" 수동 단계를 **0분**으로. 명령 한 줄 → 완성 영상. |
| **비용·난이도** | 무료(OSS) / 난이도 ↑↑ (React+영상 합성 학습). 렌더 서버/시간 필요. |
| **약점** | 정적 텍스트 위주라 **트렌드 감성 약함**(PLAN §1.2에서 비선택 사유). 손글씨·트렌드 전환은 여전히 CapCut이 우위. |
| **언제** | "메시지 콘텐츠(B타입)" 양산이 많아지고, 편집 5분조차 부담될 때. |
| **선결** | `templates/`에 Remotion 컴포지션 + 폰트, 렌더 파이프라인. |

## 후보 2. YouTube Data API — 트렌드 자동 수집

| 항목 | 내용 |
|------|------|
| **무엇** | 현재 trend-scout는 Claude 학습지식 기반(컷오프 한계). YouTube Data API로 **실시간 인기 쇼츠/키워드**를 긁어와 근거 강화. |
| **해결** | 트렌드 신선도·정확도 ↑. "지난주 실제 인기" 데이터로 적합도 점수 보정. |
| **비용·난이도** | API 무료 쿼터 있음 / 난이도 中. `.env`에 `YOUTUBE_API_KEY`. |
| **약점** | 인스타·틱톡·CapCut 트렌드는 못 가져옴(YouTube 한정). 보조 신호로만. |
| **언제** | 주간 트렌드 보고의 신뢰도를 올리고 싶을 때. `scout_trends.py`에 수집 소스로 추가. |
| **선결** | Google Cloud 프로젝트 + API 키, 쿼터 관리. |

## 후보 3. Instagram Graph API — 자동 업로드

| 항목 | 내용 |
|------|------|
| **무엇** | 완성 PNG/영상 + 캡션을 **인스타에 자동 게시**(현재는 수동 업로드). |
| **해결** | "폰으로 받아 직접 업로드" 마지막 수동 단계 제거. 예약 발행 가능. |
| **비용·난이도** | 무료 / 난이도 ↑ (비즈니스 계정·페이스북 앱 심사·토큰 갱신 번거로움). |
| **약점** | 정책·토큰 만료 관리 부담. 게시 실패 시 알림 필요. 개인 계정 불가(비즈/크리에이터 전환 필수). |
| **언제** | 게시 빈도가 늘고 예약 발행이 필요할 때. `execution/upload_to_ig.py`(현재 뼈대) 본격 구현. |
| **선결** | IG 비즈니스 계정 + FB 페이지 연결 + `INSTAGRAM_GRAPH_API_TOKEN`. |

## 추천 도입 순서 (부장 의견)

1. **YouTube API (난이도 中, 효과 즉시)** — 트렌드 정확도부터 올리는 게 비용 대비 효과 큼.
2. **Instagram Graph API** — 운영이 손에 익고 게시 빈도가 늘면.
3. **Remotion** — 영상 물량이 많아지고 편집 시간이 진짜 병목일 때 (가장 무거움).

> ⏭️ Phase 3 보류: ElevenLabs(AI 음성), HeyGen(아바타) — PLAN §1.2 비선택 사유 유지.

## 결정 방법

실전 운영 2~4주 후 `directives/learnings.md`에 쌓인 "수동 단계 병목"을 보고, 가장 아픈 곳부터 1개 선택해 별도 PLAN/CHECKLIST를 새로 끊어 진행한다(Phase 1과 동일한 단계식).
