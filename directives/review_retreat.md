# 수련회 점검 SOP

## 목표
기획안 또는 진행 중인 수련회를 단계별 체크리스트로 점검해 위험을 사전에 잡는다.

## 입력
- 점검 대상 기획안 파일 경로 (`.tmp/dossiers/*.md`)
- 점검 시점 (D-30 / D-7 / D-1 / 당일 / 사후)

## 사용할 도구
- `execution/check_retreat.py` — 체크리스트별 점검 실행
- Claude (직접 분석)

## 점검 항목

### D-30 (한 달 전)
- [ ] 강사 확정 + 사례비 송금 일정
- [ ] 장소 계약서 + 잔금 일정
- [ ] 참가자 모집 50% 이상
- [ ] 식사·교통 업체 견적 확정
- [ ] 보험 가입 (50명 이상이면 필수)

### D-7 (일주일 전)
- [ ] 참가자 명단 최종 확정
- [ ] 알레르기·복용약·특이사항 수합
- [ ] 비상연락망 (강사·교회·병원·119)
- [ ] 비상시 대피·후송 동선
- [ ] 자료집/PPT/음향 준비 완료

### D-1 (전날)
- [ ] 짐 패킹 리스트 점검
- [ ] 현장 답사 또는 도착 후 즉시 점검
- [ ] 스태프 R&R 최종 확정

### 당일
- [ ] 강사 도착 확인
- [ ] 음향·영상 리허설
- [ ] 응급의약품·구급함 위치 공지

### 사후 (D+7 이내)
- [ ] 사례비·정산 마감
- [ ] 참가자 설문 발송
- [ ] 강사·장소 후기 → `data/speakers.json`, `venues.json` 업데이트
- [ ] 학습 기록 → 이 문서 하단에 반영

## 출력
`.tmp/dossiers/{교회명}_점검_{시점}.md` 형식 보고서.
각 항목 → ✅ / ⚠️ / ❌ + 한 줄 코멘트.

## 엣지 케이스
- 체크 항목이 ❌인 경우 → 즉시 사용자에게 알림(슬랙·문자 등)
- 강사 일정 충돌 발견 → 대체 강사 자동 추천 (`data/speakers.json`에서)
